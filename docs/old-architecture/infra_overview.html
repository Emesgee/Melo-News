Melo-News: Application & Infrastructure Overview

1. Application Structure

The Melo-News project is organized as a modular Python application with supporting scripts, Docker configuration, and an Android client. Key directories and files include:

  • app/ – Main backend application (APIs, models, analytics, auth, etc.)
  • modules/ – Core logic (database, geocoder, video handler, etc.)
  • scripts/ – Utility and data processing scripts
  • data/ – GeoJSON and other datasets
  • android/ – Android client source
  • Dockerfile, docker-compose.yaml – Containerization and orchestration
  • requirements.txt – Python dependencies

2. Infrastructure Overview

The application is designed for deployment on cloud infrastructure (e.g., DigitalOcean) and local development. Key infrastructure components:

  • Docker – Containerizes the backend for consistent deployment
  • docker-compose – Orchestrates multi-service setups (API, DB, etc.)
  • nginx – Serves as a reverse proxy (see nginx.conf)
  • SSH tunnels – Used for secure local access to remote services
  • DigitalOcean – Typical cloud host for production

3. Development & Deployment Notes

  • Local development uses venv312 for Python environment management.
  • React frontend (if used) should proxy API requests via SSH tunnel to 127.0.0.1 (not localhost on Windows).
  • Deployment is managed via Docker Compose and DigitalOcean droplets.
  • See DEPLOYMENT_GUIDE.md and INSTALLATION.md for detailed steps.

4. Key Files & Directories

  • main.py – Application entry point
  • config.py – Configuration settings
  • nginx.conf – Nginx reverse proxy configuration
  • docker-compose.yaml – Service orchestration
  • app/ai_analyzer/ – AI/ML components
  • modules/ – Shared backend modules
