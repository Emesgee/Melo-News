Melo-News: Application & Infrastructure Overview

Reflects the current citizen-journalism-trust-model architecture (2026-07-17). The previous "news aggregator" version is archived under docs/old-architecture/. Canonical decisions: docs/adr/; narrative: docs/architecture.md.

What Melo-News is. A reader-side trust layer for conflict content (ADR-0019): it shows a lay reader which of tonight's many posts is corroborated, by how many independent sources, as a non-binary "basis of trust." It is not a scraper, an AI summariser, or a single-item provenance tool — those cells are already owned by better-resourced tools; stages 6→7 (corroboration → reader-legible trust) are the unclaimed one.

1. Application structure

  • app/ – Flask backend (app-factory + blueprints)
    • events/ – the trust engine: clustering, recompute_event, independence detector, corroboration graph (archive.py), snapshots
    • story/ – report ingest + serialization
    • moderation/ – verification queue & status override
    • auth/ · profile/ · identity/ – pseudonymous identity, JWT turnstile, trust ladder
    • frontend/ – React 18 web reader + moderation surface
    • drill/ – scripted pilot (T4P corroboration drill)
  • modules/ – object storage abstraction (s3_handler.py, azure_handler.py, object_storage.py)
  • android/ – Kotlin signed-reporter client (Lane B): on-device signing, EXIF strip, offline queue, panic/decoy, mesh relay
  • docs/adr/ – Architecture Decision Records (the source of truth)
  • main.py · config.py – entry point & single-source config
  • Dockerfile · docker-compose.yaml · nginx.conf – containerised deploy

2. The trust engine (the core)

The Event (an incident) is the primary reader-facing unit (ADR-0004), not a raw report. Reports (FileUpload rows) cluster into an Event by geography + time; one incident = one Event = one map pin = one shareable page. Two counts drive it:

  • corroboration_count – distinct verified, non-anonymous identities.
  • independent_source_count – distinct identities after collapsing byte-identical media to one origin (the fake-independence detector: a union-find over people; a clip reshared under many pseudonyms is one origin, not corroboration — UC3/UC8). This is the number the CORROBORATED gate and the reader display trust. Promotion also needs a rung-2+ member (Sybil backstop, ADR-0005).

Status DEVELOPING | CORROBORATED | DISPUTED | CLOSED is derived by recompute_event(). On entering an archival status it writes an append-only, content-addressed EventGraphSnapshot (capture-before-deletion, UC9). Text/timing coordination is surfaced as advisory-only signals (never reduces the count).

3. Two input lanes, one Event (ADR-0019)

LaneWhatTrust tierStatus
Lane B Signed client report — Android captures & signs on-device (ECDSA P-256, AndroidKeyStore); web /upload is a login-required unsigned variant "verified source" (premium) BUILT
Lane A Already-public post (Telegram/X link or monitored channel) → geolocate + timestamp + content-match into an Event "public source" (lower) DEFERRED (Phase 2)

A reader is meant to see "N sources: X verified, Y public" — honest about which is which. Lane A is the mass-audience path but is gated behind the pilot's belief-change result (ADR-0020).

5. Clients (ADR-0007, ADR-0021)

  • Web is the reader + moderation surface — the primary product, no install, where the belief question is tested. Reader browse is login-free; reporter/moderator actions require login.
  • Android is reporter-first — the hardened signed capture path. It carries only a thin, decoy-gated awareness layer (own track record, compromise alert, a situational-awareness event map). It must not grow into a full reader browser — the reporter's phone is the highest-risk object under seizure, so it holds the minimum.

6. Storage & infrastructure (ADR-0017)

  • Object storage – clients upload media directly to a private S3-compatible bucket via presigned PUT; readers get short-lived presigned GET URLs. The server never proxies media bytes. STORAGE_BACKEND selects s3 (Hetzner/MinIO) or azure (legacy).
  • Database – PostgreSQL via SQLAlchemy (pooled, pre-ping/recycle).
  • Deploy – containerised (Dockerfile, compose) behind nginx on a Hetzner host. Dev phone connectivity during the drill is over Tailscale.

7. Identity & trust

  • Device keypair = pseudonym (ADR-0003/0013): the ECDSA P-256 public key IS the identity (AndroidKeyStore, hardware-backed). No email/password for reporters.
  • JWT is only a turnstile (ADR-0016); the signature is the real identity. Reader-side media verification rebuilds the canonical signed message (ADR-0009/0014).
  • Trust ladder rung 1..3 (0 = anonymous, no user row). Earned only via corroborated history + time, or a steward vouch. Roles: reporter · moderator · steward.

8. Security posture

Hard release gate (ADR-0011). Real at-rest encryption, Keystore-backed secrets, and signature-only auth are hard-blocking before any real at-risk reporter uses the app. That layer is currently stubbed — the app must not be presented as safe for real field journalists until ADR-0011 is done. The current pilot runs on signed pseudonyms with a dummy cohort (the drill).
  • Anonymous public ingest is disabled for the pilot (easiest Sybil/spam vector; counts 0 toward corroboration anyway).
  • Web-account passwords are hashed (irrecoverable, only resettable).
  • Lane A readers install no capture client, so there is no on-device identity to seize — de-risking the mass case (ADR-0019); ADR-0011 still gates Lane B.

9. Build sequence (ADR-0020)

  1. Phase 0 — the drill (current): scripted T4P corroboration exercise, re-scoped to falsify the belief-change hypothesis. Gates everything after.
  2. Phase 1 — no-regret core (BUILT, 135 tests green): archive-grade data model, fake-independence detector, corroboration graph, durable snapshots, advisory signals.
  3. Phase 2 — branch on the drill: strong belief-change → build Lane A ingest + graded feed; weak/backfire → emphasise archive and/or debunk shapes.
  4. Phase 3 — per-surface hardening: ADR-0011 before any real Lane-B reporter; moderation-at-scale + takedown/jurisdiction before any public Lane-A launch.
